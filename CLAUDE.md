# Coding Style

Write code in a minimalist research-engineer style inspired by Andrej Karpathy.

Principles:
- readability over cleverness
- educational clarity over abstraction
- self-contained implementations
- explicit tensor/math operations
- minimal dependencies
- comments explain why, not what; explain every single line of block code. 

Avoid:
- enterprise patterns
- unnecessary abstractions
- premature optimization
- factory patterns
- deep inheritance trees

Code should:
- mirror the mathematics directly
- be understandable in one sitting
- prioritize transparency over extensibility
- feel like a clean research notebook

Prefer:
- concise files
- explicit training loops
- direct data flow
- descriptive variable names
